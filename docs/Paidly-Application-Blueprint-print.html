Paidly Architecture v2 (clean reference)

One-page upgrade summary. The Part A outline below also names Relationship & Offering separatelyвҖ”the client/catalog input graph that feeds the Document Engine.

Paidly is structured around four core engines:

  1. Identity System вҖ” Auth, users, organizations, roles, RLS-backed tenancy.
  2. Document Engine вҖ” Invoices, quotes, payslips unified as document(type=вҖҰ); shared compose, send, and PDF paths.
  3. Financial Engine вҖ” Payments, cash flow, reports, subscriptions (reads and rails downstream of documents).
  4. Growth Engine вҖ” Affiliates, email, notifications, CRM-style behaviour (how Paidly scales).

Frontend layout system

Same structure everywhere:

  • Header вҖ” title + primary actions
  • Content вҖ” tables, grids, main forms
  • Side panel вҖ” summary, filters, secondary controls (PageTemplate in code)

Data flow (client):

UI вҶ’ Hooks вҶ’ Services вҶ’ Entity facades (EntityManager) вҶ’ Supabase

Serverless APIs (/api/*) handle:

  • Payments (Payfast)
  • Emails (Resend / SMTP)
  • Public document access (shares, tokens)
  • Cron jobs (dunning, reminders, ops)
Goal: Evolve Paidly from an invoicing tool into a full business operating system.

Part A вҖ” Core systems

1. Identity & Access

Supabase Auth, profiles, orgs, memberships, roles, RLS. Strategy: stabiliseвҖ”harden session, org bootstrap, RequireAuth so other systems trust it cheaply.

2. Document Engine (commerce core)

Job: Issue documents to third parties вҶ’ deliver вҶ’ track вҶ’ settle. Kinds today: Invoice, Quote, Payslip (each maps to existing entities/tables).

Shared lifecycle: Author вҶ’ Compose вҶ’ Render (PDF) вҶ’ Deliver (email/link) вҶ’ Observe (opens, reminders) вҶ’ Settle (pay / accept / archive).

Product point: Invest once in send, PDF, public token, status, and line-item patternsвҖ”parity across kinds is engine work, not three copies.

3. Relationship & Offering

Clients + catalog (services) + line intelligenceвҖ”input graph into the Document Engine, not a disconnected вҖңcontacts app.вҖқ

4. Revenue & Operating

Payfast, subscriptions, crons, affiliates, admin, reporting. Strategy: keep thin on rules that belong inside documents (e.g. overdue logic with invoices).
