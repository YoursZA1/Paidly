Production-ready

Hybrid
  • Audit Log (`/admin-v2/audit-log`) — server table when available, local/unified fallback when not.
  • Platform users loader — API-first with fallback to profile/entity list for non-auth/config failures.
  • Affiliate loader model — backend-driven queue with mixed local/entity surroundings.
Needs backend wiring
  • Settings command layer (`/admin-v2/settings`) — major controls persist to browser/local storage.
  • System Health chips — currently presentation-level labels, not probe-driven service health.
  • Activity Log card — local audit stream + fallback entries, not authoritative server feed first.
  • Danger Zone actions — UI/local operational changes, not full backend workflow enforcement.
