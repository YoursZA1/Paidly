# Email Confirmation Template Setup

This guide explains how to set up the branded email confirmation template for Paidly.

## Files Included

- `EMAIL_CONFIRMATION_TEMPLATE.html` – HTML version (recommended for best appearance)
- `EMAIL_CONFIRMATION_TEMPLATE.txt` – Plain text version (fallback)

## How to Use

### 1. Open Supabase Dashboard

Go to your Supabase project → **Authentication** → **Email Templates**

### 2. Find "Confirm Signup"

In the Email Templates section, click on **"Confirm Signup"** to edit it.

### 3. Copy the HTML Template

Open `EMAIL_CONFIRMATION_TEMPLATE.html` and copy the entire HTML content (everything between `<html>` and `</html>`).

### 4. Paste into Supabase

In the Supabase Email Templates editor:

- **Subject:** 
  ```
  Confirm your email for Paidly
  ```

- **Template Body:** 
  Paste the HTML template you copied

### 5. Enable Email Confirmations (if not already done)

In **Authentication → Providers → Email:**
- Toggle **Email confirmations** to **ON**
- Set confirmation email expiry (default 24 hours is fine)

### 6. Configure URL Settings

In **Authentication → URL Configuration:**
- Set **Site URL** to your production domain (e.g., `https://paidly.vercel.app`)
- Add Redirect URLs:
  - `https://paidly.vercel.app/login`
  - `https://paidly.vercel.app/auth/callback`
  - `http://localhost:5173/login` (for local dev)

## Template Variables

The template uses these Supabase variables (automatically populated):

- `{{ .Email }}` – User's email address
- `{{ .ConfirmationURL }}` – The confirmation link (auto-generated by Supabase)
- `{{ .SiteURL }}` – Your Site URL (if needed, add to template)

## Customization

You can customize the template further:

- **Colors:** Change `#f24e00` (orange) to your brand color
- **Company Info:** Update the footer with your actual address and support email
- **Logo:** Replace the 🧡 emoji with an actual logo by adding an image tag:
  ```html
  <img src="https://yourcdn.com/logo.png" alt="Paidly Logo" style="width: 60px; height: auto;">
  ```
- **Text:** Update messaging to match your brand voice

## Testing

1. **Local Development:**
   - Disable email confirmations temporarily (for quick testing)
   - Or use a test email service like Mailtrap

2. **Production Testing:**
   - Create a new test account with a real email you control
   - Verify the email arrives with correct formatting
   - Click the confirmation link and ensure it redirects properly

## Troubleshooting

### Emails not sending
- Check **Supabase Logs** for auth errors
- Verify `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY` are set correctly in Vercel
- Ensure email is from a verified sender in Supabase

### Template not rendering correctly
- Clear browser cache and test again
- Ensure all HTML is properly formatted (no extra line breaks in critical sections)
- Test with different email clients (Gmail, Outlook, etc.)

### Users not receiving email
- Check spam/junk folder
- Verify Site URL matches your actual domain
- Check Supabase Dashboard → Logs for errors
